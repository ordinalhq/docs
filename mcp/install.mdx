## Overview

The Ordinal MCP server is an OAuth-protected Streamable HTTP endpoint. Modern clients connect with just the URL — they handle the OAuth 2.1 sign-in flow for you.

<Card title="MCP Server URL" icon="server">
  ```
  https://app.tryordinal.com/mcp
  ```
</Card>

<Info>
  **No API keys needed.** When you first connect, your client opens a browser tab for you to sign in with your Ordinal account. The access token is stored and refreshed transparently. See [Authentication](/mcp/authentication) for the protocol details.
</Info>

## Install in your client

<Tabs>
  <Tab title="Claude Desktop">
    Open Claude Desktop settings (`Cmd + ,` on Mac, `Ctrl + ,` on Windows), go to **Connectors**, click **Add custom connector**, and paste the MCP server URL:

    ```
    https://app.tryordinal.com/mcp
    ```

    Claude Desktop will open a browser window for the OAuth sign-in. After signing in with your Ordinal account, the connector shows **Connected** and the Ordinal tools become available in chat.
  </Tab>

  <Tab title="Claude Code">
    Add the server with a single command:

    ```bash
    claude mcp add --transport http ordinal https://app.tryordinal.com/mcp
    ```

    Claude Code will prompt you to sign in on first use. Verify with:

    ```bash
    claude mcp list
    ```

    You should see `ordinal` listed as connected.
  </Tab>

  <Tab title="Cursor">
    Open Cursor settings, go to **MCP**, and add a new server. Cursor supports OAuth-protected MCP servers natively. Configure the server with:

    ```json
    {
      "mcpServers": {
        "ordinal": {
          "url": "https://app.tryordinal.com/mcp"
        }
      }
    }
    ```

    Save and restart Cursor. Use the command palette (`Cmd+Shift+P`) → "Open MCP settings" to trigger the sign-in flow.
  </Tab>

  <Tab title="VS Code">
    Create (or edit) `.vscode/mcp.json` at the root of your project:

    ```json
    {
      "servers": {
        "ordinal": {
          "type": "http",
          "url": "https://app.tryordinal.com/mcp"
        }
      }
    }
    ```

    Reload the window. VS Code's MCP integration will prompt you to sign in the first time a Copilot agent calls an Ordinal tool.
  </Tab>

  <Tab title="Gemini CLI">
    Add the server via CLI:

    ```bash
    gemini mcp add --transport http ordinal https://app.tryordinal.com/mcp
    ```

    Or edit `~/.gemini/settings.json` directly:

    ```json
    {
      "mcpServers": {
        "ordinal": {
          "httpUrl": "https://app.tryordinal.com/mcp"
        }
      }
    }
    ```

    Verify with `gemini mcp list`.
  </Tab>

  <Tab title="Codex CLI">
    Add to `~/.codex/config.toml`:

    ```toml
    [mcp_servers.ordinal]
    url = "https://app.tryordinal.com/mcp"
    ```

    Verify with `/mcp` in the Codex TUI.
  </Tab>
</Tabs>

## Clients without native OAuth

If your MCP client only supports stdio transport or doesn't speak OAuth, you can bridge through [`mcp-remote`](https://www.npmjs.com/package/mcp-remote). It handles the OAuth handshake locally and proxies MCP messages over stdio:

```json
{
  "mcpServers": {
    "ordinal": {
      "command": "npx",
      "args": ["mcp-remote", "https://app.tryordinal.com/mcp"]
    }
  }
}
```

`mcp-remote` opens a browser for the first sign-in and stores tokens under `~/.mcp-auth/`. No API keys, no pasted headers.

## Verifying the connection

Ask the assistant to list your Ordinal workspaces. It should call `ordinal_get_workspace_context` with no arguments and return a list of workspaces you can access. If the call fails with `-32001` or `-32002`, see [Authentication → Troubleshooting](/mcp/authentication#troubleshooting).

## Upgrading from the v1 MCP server

The previous server at `https://app.tryordinal.com/api/mcp` authenticated with a bearer API key. It has been replaced by the OAuth-based server at `https://app.tryordinal.com/mcp`.

<Warning>
  **Action required:** Update your MCP client configuration to the new URL. Remove any `Authorization: Bearer ord_...` headers — they are no longer used. On first connect, the client will prompt you to sign in with your Ordinal account.
</Warning>

Key differences:

- **URL:** `/api/mcp` → `/mcp`
- **Auth:** workspace API key → per-user OAuth 2.1 access token
- **Scope:** single workspace → every workspace your user belongs to (pass `workspaceSlug` per tool call)
- **Tool names:** the v2 tools are renamed and restructured — e.g., post CRUD is now split between `ordinal_create_post`, `ordinal_manage_post`, and `ordinal_search_content`. See the [Tools reference](/mcp/tools).
