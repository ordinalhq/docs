## Overview

The MCP server exposes **22 tools** across 8 categories. MCP clients fetch full JSON Schemas at runtime via the `list_tools` request â€” this page is a human-readable overview so you can understand the surface area without connecting.

All tools share a few conventions:

- **`workspaceSlug`** is required on every tool except the initial discovery call. Get it from `ordinal_get_workspace_context` with no arguments.
- **`responseFormat`** can be `"json"` (default) or `"markdown"` for human-readable output.
- **Tool annotations** (`readOnlyHint`, `destructiveHint`, `idempotentHint`, `openWorldHint`) help the assistant reason about when to call a tool.

<Info>
  **Start every session with `ordinal_get_workspace_context`.** It returns the IDs (profiles, labels, users, campaigns) every other tool needs.
</Info>

## Discovery

| Tool | Purpose |
|------|---------|
| `ordinal_get_workspace_context` | Bootstrap a session. Omit `workspaceSlug` to list accessible workspaces; pass it for full context â€” scheduling profiles, engagement profiles, labels, users, workspace, campaigns. |

## Content

| Tool | Purpose |
|------|---------|
| `ordinal_search_content` | Search posts or ideas with filters (status, labels, campaigns, date range) or fetch a single post/idea by `postId` with comments. |
| `ordinal_create_post` | Create a post. Requires `title`, `publishAt` (ISO 8601), and at least one channel config (`linkedIn`, `x`, or `instagram`). Optional: `status`, `labelIds`, `campaignId`, `notes`. |
| `ordinal_create_idea` | Create an idea. Same channel-shaped payload as `ordinal_create_post`; no `publishAt` required. |
| `ordinal_manage_post` | Update post metadata or channel content, archive, or unarchive a post. |
| `ordinal_manage_idea` | Update idea metadata or channel content, archive, unarchive, or add idea to calendar. |

Content mutation tools use **channel-shaped inputs**: keep shared fields (`workspaceSlug`, `title`, `publishAt`, `status`, `labelIds`, `campaignId`, `notes`) at the top level and put channel-specific content inside `linkedIn`, `x`, or `instagram`.

## Assets

| Tool | Purpose |
|------|---------|
| `ordinal_create_upload` | Start an async upload job for an image or video URL. Returns an upload job ID. |
| `ordinal_get_upload` | Poll an upload job by ID. Returns status and the resulting `assetId` when ready. |

## Analytics

| Tool | Purpose |
|------|---------|
| `ordinal_get_analytics` | Retrieve post performance or follower growth for LinkedIn, X, Instagram, or Facebook. Pass `platform`, `type` (`"posts"` or `"followers"`), `profileId`, and an optional date range (defaults to the last 30 days). |

## Collaboration

| Tool | Purpose |
|------|---------|
| `ordinal_list_approvals` | List approval requests on a post â€” who approved, who is still pending, due dates. |
| `ordinal_manage_approvals` | Create or delete approval requests on a post. |
| `ordinal_list_subscribers` | List subscribers notified about a post. |
| `ordinal_manage_subscribers` | Add or remove subscribers on a post. |
| `ordinal_manage_comments` | Create a comment (supports `@mentions`) or delete one of your own comments. Reading comments is handled by `ordinal_search_content` with `postId`. |

## Auto-engagements

| Tool | Purpose |
|------|---------|
| `ordinal_list_engagements` | List auto-engagements (Like, Comment, Repost) attached to a post, including their delay and target channel. |
| `ordinal_manage_engagements` | Create, update, or delete auto-engagements. `delaySeconds` must be one of the allowed values (0, 300, 600, 1800, 3600, 14400, 28800, 43200, 86400). Comments require `copy`; Likes do not. |

## Slack Boosts

| Tool | Purpose |
|------|---------|
| `ordinal_list_slack_boosts` | List Slack boosts on a post, or fetch a single boost by `boostId`. |
| `ordinal_manage_slack_boosts` | Create, update, or delete Slack boosts tied to a post. |

## Workspace

| Tool | Purpose |
|------|---------|
| `ordinal_manage_labels` | Create or delete labels on a workspace. Label colors are restricted to a fixed palette (`yellow`, `purple`, `orange`, `red`, `brown`, `green`). |

## Platform utilities

| Tool | Purpose |
|------|---------|
| `ordinal_get_linkedin_mention` | Look up a LinkedIn user or organization by username/vanity name. Returns URN and mention format strings for use in post copy. |
| `ordinal_get_linkedin_profile` | Look up a LinkedIn profile by URN. |
| `ordinal_search_instagram_locations` | Search Instagram location tags for use in Instagram posts. |

## Response formats

All tools accept an optional `responseFormat` argument:

- `"json"` â€” structured JSON (default). Best for programmatic consumers.
- `"markdown"` â€” human-readable Markdown. Useful when you want the assistant's reply to embed the result directly.

## Errors

Tool errors come back as structured MCP tool errors (`isError: true` with a textual `content` payload), not as JSON-RPC errors. Common cases:

- **Validation errors** include the failing field and constraint (e.g. "At least one channel config is required").
- **Not found / unauthorized** on `workspaceSlug` indicates either a typo or a workspace the authenticated user can't access.
- **401 at the transport level** (not a tool error) means the OAuth access token is missing or expired â€” see [Authentication](/mcp/authentication).
