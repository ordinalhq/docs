## Overview

Inline comments are text-anchored comments attached to specific content within a post. Unlike regular post comments, inline comments highlight specific text and are organized by channel (LinkedIn, Twitter, etc.).

The API provides read-only access to inline comments. Creating and resolving inline comments is done through the Ordinal web interface.

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/posts/{postId}/inline-comments` | [List inline comments on a post](/api-reference/inline-comments/inline-comments-list) |

## Listing Inline Comments

Retrieve all inline comments across all channels on a post:

```bash
curl -X GET "https://app.tryordinal.com/api/v1/posts/POST_ID/inline-comments" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `channel` | string | No | Filter by channel (see supported channels below) |
| `resolved` | boolean | No | Filter by resolution status (`true` or `false`) |

### Filter by Channel

```bash
curl -X GET "https://app.tryordinal.com/api/v1/posts/POST_ID/inline-comments?channel=LinkedIn" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Filter by Resolution Status

```bash
# Get only unresolved comments
curl -X GET "https://app.tryordinal.com/api/v1/posts/POST_ID/inline-comments?resolved=false" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Combine Filters

```bash
curl -X GET "https://app.tryordinal.com/api/v1/posts/POST_ID/inline-comments?channel=LinkedIn&resolved=false" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Supported Channels

| Channel | Value |
|---------|-------|
| LinkedIn | `LinkedIn` |
| X (Twitter) | `Twitter` |
| Instagram | `Instagram` |
| Facebook | `Facebook` |
| Threads | `Threads` |
| TikTok | `TikTok` |
| Discord | `Discord` |
| Slack | `Slack` |
| Webflow | `Webflow` |
| YouTube Shorts | `YouTubeShorts` |

## Inline Comment Object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier for the inline comment thread |
| `resolved` | boolean | Whether the comment thread has been marked as resolved |
| `channel` | string | The channel this comment is on (LinkedIn, Twitter, etc.) |
| `highlightedText` | string | The text that was highlighted when the comment was created |
| `replies` | array | Array of replies in the thread (see Reply Object) |
| `createdAt` | string (ISO 8601) | When the thread was created |
| `updatedAt` | string (ISO 8601) \| null | When the thread was last updated |

## Reply Object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier for the reply |
| `user` | object | The user who wrote the reply |
| `user.id` | string (UUID) | User's unique identifier |
| `user.firstName` | string \| null | User's first name |
| `user.lastName` | string \| null | User's last name |
| `user.email` | string | User's email address |
| `message` | string | The reply text, with mentions in `@[Name](id)` format |
| `createdAt` | string (ISO 8601) | When the reply was created |
| `reactions` | array | Array of emoji reactions (see Reaction Object) |

## Reaction Object

| Field | Type | Description |
|-------|------|-------------|
| `emoji` | string | The emoji used for the reaction |
| `users` | array | Array of users who added this reaction |

## Understanding Inline Comments

### Highlighted Text

The `highlightedText` field contains the exact text that was selected when the comment was created. This helps provide context even if the post content has since changed.

<Note>
If the highlighted text has been deleted or significantly modified, the `highlightedText` field may be empty or may not match the current post content.
</Note>

### Resolution Status

Inline comments can be marked as resolved when feedback has been addressed:
- `resolved: false` - Comment still needs attention
- `resolved: true` - Comment has been addressed

Use the `resolved` query parameter to filter comments by status.

### Threaded Replies

Each inline comment contains an array of `replies`. The first reply is the original comment, and subsequent replies form a threaded conversation. Comments with no non-deleted replies are not included in the response.

## Error Responses

| Status | Code | Description |
|--------|------|-------------|
| 404 | `NOT_FOUND` | Post not found or doesn't belong to the workspace |

```json
{
  "code": "NOT_FOUND",
  "message": "Post not found or does not belong to this workspace"
}
```

## Important Notes

<Note>
The inline comments API is read-only. To create, reply to, or resolve inline comments, use the Ordinal web interface.
</Note>

<Note>
Inline comments are stored per-channel. A post with LinkedIn and Twitter content will have separate inline comments for each channel.
</Note>
