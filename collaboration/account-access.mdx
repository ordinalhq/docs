## Overview

Control who can access what in your Ordinal workspace. Set up roles, permissions, and access levels to keep your accounts secure.

## User Roles

### Owner

Full control over the workspace:
- Manage billing and subscription
- Add/remove team members
- Delete workspace
- All other permissions

### Admin

Administrative access:
- Manage team members
- Configure settings
- Connect/disconnect profiles
- Manage all content

### Member

Standard team member:
- Create and edit posts
- Schedule content
- View analytics
- Comment and collaborate

### Viewer

Read-only access:
- View posts and calendar
- View analytics
- Cannot create or edit content

## Managing Team Members

### Invite Team Members

1. Go to **Settings** → **Team**
2. Click **Invite Member**
3. Enter email address
4. Select role
5. Send invitation

### Change Roles

Update a team member's role:
1. Go to **Settings** → **Team**
2. Find the team member
3. Click their role
4. Select new role
5. Save changes

### Remove Team Members

Remove access:
1. Go to **Settings** → **Team**
2. Find the team member
3. Click **Remove**
4. Confirm removal

## Profile-Level Permissions

Control access to specific profiles:

| Permission | Description |
|------------|-------------|
| **Full Access** | Create, edit, publish, delete |
| **Can Edit** | Create and edit, cannot publish |
| **Can View** | View only, no editing |
| **No Access** | Profile hidden from user |

## Security Best Practices

<AccordionGroup>
  <Accordion title="Least privilege principle">
    Give team members only the access they need for their role.
  </Accordion>
  <Accordion title="Regular audits">
    Review team access quarterly and remove unused accounts.
  </Accordion>
  <Accordion title="Offboarding">
    Remove access immediately when team members leave.
  </Accordion>
</AccordionGroup>
