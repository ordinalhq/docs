---
title: "post.permanently_deleted"
description: "Fired when a post is permanently deleted"
sidebarTitle: "post.permanently_deleted"
---

All webhook payloads follow a consistent top-level structure with event-specific data nested within the `data` object.

<ResponseField name="type" type="string" required>
  The event type (e.g. `post.permanently_deleted`).
</ResponseField>

<ResponseField name="data" type="object" required>
  Event-specific payload for this webhook.

  <Expandable title="Data object parameters">
    <ResponseField name="id" type="string" required>
      Post ID.
    </ResponseField>

    <ResponseField name="title" type="string" required>
      Post title.
    </ResponseField>

    <ResponseField name="status" type="string" required>
      Post status.
    </ResponseField>

    <ResponseField name="channels" type="array" required>
      Channels the post was scheduled for.
    </ResponseField>

    <ResponseField name="campaign" type="object | null">
      Campaign if assigned. Contains `id` and `name`.
    </ResponseField>

    <ResponseField name="url" type="string" required>
      Ordinal post URL.
    </ResponseField>

    <ResponseField name="workspace" type="object" required>
      Workspace the post belonged to. Contains `id`, `slug`, and `name`.
    </ResponseField>

    <ResponseField name="deletedBy" type="object | null">
      User who deleted the post, or null if auto-deleted after trash retention. Contains `id`, `firstName`, `lastName`, and `email`.
    </ResponseField>

    <ResponseField name="deletedAt" type="string" required>
      ISO 8601 timestamp when the post was permanently deleted.
    </ResponseField>

    <ResponseField name="linkedIn" type="object | null">
      LinkedIn content when the post included LinkedIn. Contains `profile` (id, name, detail), `copy`, and `assets` (id, name, url, mimetype, size, width, height, duration).
    </ResponseField>

    <ResponseField name="x" type="object | null">
      X/Twitter content when the post included Twitter. Contains `profile` (id, name, detail) and `tweets` (array of `{ copy, assets }`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp when the webhook event was created.
</ResponseField>
