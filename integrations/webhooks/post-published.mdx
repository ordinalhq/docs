---
title: "post.published"
description: "Fired when a post is successfully published to a social channel"
sidebarTitle: "post.published"
---

All webhook payloads follow a consistent top-level structure with event-specific data nested within the `data` object.

<ResponseField name="type" type="string" required>
  The event type (e.g. `post.published`).
</ResponseField>

<ResponseField name="data" type="object" required>
  Event-specific payload for this webhook.

  <Expandable title="Data object parameters">
    <ResponseField name="id" type="string" required>
      Post ID.
    </ResponseField>

    <ResponseField name="title" type="string" required>
      Post title.
    </ResponseField>

    <ResponseField name="channel" type="string" required>
      Channel where the post was published (e.g. LinkedIn, Twitter).
    </ResponseField>

    <ResponseField name="campaign" type="object | null">
      Campaign if the post is assigned to one. Contains `id` and `name`.
    </ResponseField>

    <ResponseField name="url" type="string" required>
      Ordinal post URL.
    </ResponseField>

    <ResponseField name="postUrl" type="string | null">
      Published URL on the social platform.
    </ResponseField>

    <ResponseField name="profile" type="object" required>
      Profile that published the post. Contains `id`, `name`, and `detail`.
    </ResponseField>

    <ResponseField name="workspace" type="object" required>
      Workspace the post belongs to. Contains `id`, `slug`, and `name`.
    </ResponseField>

    <ResponseField name="publishedBy" type="object" required>
      User who triggered the publish. Contains `id`, `firstName`, `lastName`, and `email`.
    </ResponseField>

    <ResponseField name="publishedAt" type="string" required>
      ISO 8601 timestamp when the post was published.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp when the webhook event was created.
</ResponseField>
