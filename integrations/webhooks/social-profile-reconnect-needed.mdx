---
title: "social_profile.reconnect_needed"
description: "Fired when a social profile needs to be reconnected (e.g. token expired, permissions revoked)"
sidebarTitle: "social_profile.reconnect_needed"
---

All webhook payloads follow a consistent top-level structure with event-specific data nested within the `data` object.

<ResponseField name="type" type="string" required>
  The event type (e.g. `social_profile.reconnect_needed`).
</ResponseField>

<ResponseField name="data" type="object" required>
  Event-specific payload for this webhook.

  <Expandable title="Data object parameters">
    <ResponseField name="profile" type="object" required>
      The profile that needs reconnecting. Contains `id`, `name`, `detail`, `channel`, and `profileImageUrl`.
    </ResponseField>

    <ResponseField name="workspace" type="object" required>
      Workspace the profile belongs to. Contains `id`, `slug`, and `name`.
    </ResponseField>

    <ResponseField name="detectedAt" type="string" required>
      ISO 8601 timestamp when the reconnect need was detected.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp when the webhook event was created.
</ResponseField>

<ResponseExample>

```json Example
{
  "type": "social_profile.reconnect_needed",
  "data": {
    "profile": { "id": "prof_1", "name": "Company Page", "detail": "LinkedIn", "channel": "LinkedIn", "profileImageUrl": "https://..." },
    "workspace": { "id": "ws_1", "slug": "my-workspace", "name": "My Workspace" },
    "detectedAt": "2025-02-26T09:00:00.000Z"
  },
  "createdAt": "2025-02-26T09:00:00.000Z"
}
```

</ResponseExample>
